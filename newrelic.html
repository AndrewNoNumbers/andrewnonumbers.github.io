New Relic

Increasing reader feedback and resolving navigation issues on New Relic's high-traffic docs site

Hello! This was short-term engagement I did with the docs team. As an area without a design system, the docs work required a lot of novel interaction and visual design, and also allowed me to use my experience with responsive web to ensure docs works for desktop, tablet, and mobile readers. I haven't built this project into a case study yet; however, the basics are below – please messsage me to learn more! I have a presentation ready.

What is New Relic?

New Relic is the industry-leading all-in-one observability platform, designed to allow software developers to respond faster, optimize better, and build more perfect software.

Problem

There were two main problems I was briefed on by the docs team. 1) Docs writers rely on feedback to know where to dedicate more resources, however the Docs site did not receive much feedback through its on-page feedback mechanism. 2) A UX metrics project showed that the Docs site scores low in 'Ease of use' and 'Ease of navigation' – how might we improve these scores?

Action

The first piece, getting more reader feedback for docs, had a very tight turnaround, and was a good opportunity for me to build trust with the team. By delivering a solution that was easily doable and took into account technical constraints, I was able to build a relationship with the Docs engineering team and Docs writers.

The second piece, bumping up 'Ease of use' and 'Ease of navigation', was a more involved project that involved synthesizing research data, aligning the team on the main challenges, and coming up with a holistic navigation redesign that solved the myriad of navigation flaws while aligning the new Docs design with our design system.

Result

Though there's always more to improve, this project exceeded the team's objectives. The redesigned feedback mechanism gathered much more reader feedback than the old one, allowing the Docs writer to be more agile in improving content, and also allowed the team to receive text suggestions. The navigation redesign resolved some longstanding complaints, set up the site for scale, and improved 'Ease of use' and 'Ease of navigation' scores the next time we ran the UX metrics survey.
